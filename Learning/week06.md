2. RULE2: 단순 복붙은 거절, 모든 코드에 이론적 근거를 바탕으로 주석 달기
3. RULE3: 직접검증 (AI의 답을 맹신하지 말고, 배운지식과 도구로 직접 교차 검증하기)

## AI활용 윤리와 학문적 정직성
올바른 AI 활용을 위한 3대 핵심 원칙

1. 정보의 투명성: 숨기지 않고 명확하게 밝히기
2. 저작권 및 책임: 최종 책임은 사용자 본인에게 존재
3. 기술적 종속 경계: AI에 끌려가지 않고 주도권 쥐기
   - AI가 제시한 답변의 무비판적 수용 태도 지양
   - 스스로 논리를 검증하고 자체적인 해답도출

## Cursor AI
- AI기반 코드 에디터
- VS CODE기반 확장형 개발 환경
- 프로젝트 전체 문맥을 이해하는 LLM통합

[기능]
- 코드 생성 및 설명
- 버그 분석 및 리팩토링 제안
- 테스트 코드 생성
- 문서화 자동 생성
- AI가 작성한 코드는 무조건적 '검증대상'

[핵심기능]
1. Inline Edit (즉석코드 생성) Ctrl + K, Ctrl + I
   - 코드 창에서 즉석으로 코드 생성 및 수정 요청
2. AI chat (코드 대화 및 튜터링)
   - 코드의 원리 파악 및 질문 (1:1 튜터 역할)
3. AI에게 참고할 정보를 명시적으로 지정하는 기능

## Antigravity
- 개발자의 생산성을 위한 차세대 AI 코딩 어시스턴트
- 코드를 대신 작성하는 도구가 아니라, 사고를 확장하는 도구임
  1. 자율 계획: 작업 리스트 사전 설계
  2. 도구 활용: 터미널 명령어 실행 및 패키지 설치
  3. 자기 수정: 에러 로그 분석 후 자체 코드 수정
